# SPEC — «До суда 15 минут»

Unity-прототип 3D-аркады-раннера для ПК. Юрист опаздывает на заседание и бежит
через офис, город и здание суда. Вся графика процедурная (примитивы + материалы,
без внешних ресурсов), UI на русском, управление на старом Input Manager.

## Технологии
- Unity (Built-in Render Pipeline; материалы автоматически переключаются на URP/Lit,
  если проект открыт с установленным URP — см. `MaterialFactory`).
- C# (синтаксис уровня C# 5 для максимальной совместимости), namespace `CourtRunner15`.
- UnityEngine.UI.Text + встроенный шрифт (LegacyRuntime/Arial, fallback на системный Arial) — TextMeshPro не требуется.
- Ввод: `Input.GetKey/GetKeyDown` (старый Input Manager, без настройки осей).
- Звук: AudioClip генерируются кодом (pickup, hit, win, lose, click, whoosh, shield).

## Ядро механики
- Персонаж визуально бежит вперёд; фактически мир (сегменты, препятствия, бусты)
  движется навстречу по -Z. Игрок стоит на z=0.
- 3 дорожки по X: −2.4 / 0 / +2.4. Смена полосы ~0.15 с (сглаженная).
- Прыжок: v₀=8.2, g=−26 (апекс ~1.3 м) — для низких препятствий.
- Подкат: 0.55 с, хитбокс 1.8 м → 0.7 м, наклон корпуса — для верхних препятствий.
  Подкат в воздухе = быстрый прижим к земле.
- Столкновение: −10…25% уверенности (по уровню), сбивает комбо, вспышка + тряска
  камеры + звук, 0.9 с неуязвимости (мигание).
- Столкновения считаются вручную по AABB (без зависимостей от физики);
  коллайдеры-триггеры на объектах присутствуют.

## Уверенность (0–100%)
- Старт уровня: 60%.
- Буст: +8…15% и +100…300 очков.
- ≥75% — ускорение +25% скорости, шире FOV, линии скорости, усиленный след.
- ≥90% — «Защита»: первая ошибка не снимает уверенность, но снимает защиту
  (повторно активируется при ≥90% не раньше чем через 4 с).
- 0% — поражение (экран с причиной).

## Очки
- Дистанция: +2 очка за метр × множитель.
- Буст: +100…300 × множитель.
- Комбо: +1 за буст без столкновений; множитель x1 (0–4), x2 (5–9), x3 (10+).
- Столкновение сбрасывает комбо.
- Итог уровня: очки + бонус за остаток уверенности (уверенность × 40).

## Управление
A/← — влево, D/→ — вправо, Space/W/↑ — прыжок, S/↓/Shift — подкат,
Esc — пауза/назад, R — перезапуск уровня. На выборе уровня: 1/2/3 — быстрый
запуск уровня (тестовый обход блокировки), кнопка «Открыть все уровни».

## Уровни
| # | Название | Дистанция | Скорость | Волны | Таймер |
|---|----------|-----------|----------|-------|--------|
| 1 | Офис за 15 минут до суда | 1250 м | 15 | 1.2 с | 15:00 |
| 2 | Реестр и квартира | 1350 м | 17.5 | 1.0 с | 16:40 |
| 3 | Коридоры суда | 1500 м | 20 | 0.85 с | 09:30 |

Таймер = остаток времени пропорционален остатку дистанции (успеваем ровно к финишу).

### Уровень 1 «Офис» — тёплый свет, зеленовато-серый глянцевый пол
Препятствия: стопка срочных папок (обход), «ДЕДЛАЙН!» (обход), разлитый кофе
(прыжок), голосовое 05:47 (прыжок), правки перед заседанием (подкат),
забытые приложения (прыжок).
Бусты: доверенность, кофе, папка дела, спокойный звонок, план выступления.
Декор: столы, мониторы, принтеры, растения, стеклянные переговорки, таблички
«Дело не ждёт», «Закон на нашей стороне», «Зал суда 15 минут», дальняя вывеска «СУД».

### Уровень 2 «Реестр и квартира» — дневной город, шире офиса
Препятствия: барьер «Мошенничество обнаружено» (обход), лужа «Слухи» (прыжок),
«Подтвердить перевод?» (обход), провода мошенников (прыжок), стопки договоров
(обход), журналисты со штангой (подкат), банкомат «ОШИБКА» (обход).
Бусты: выписка ЕГРН, ключи от квартиры, нотариальный договор, правовой довод,
спокойный клиент.
Декор: здания с окнами, вывески «Росреестр», «МФЦ», «Получить выписку онлайн»,
«Квартира с двойной продажей?», медиа-фургон, банкоматы, киоски, барьеры, журналисты.

### Уровень 3 «Коридоры суда» — мрамор, дерево, золото, самый быстрый
Препятствия: очередь (обход), рамка безопасности (подкат), пристав (обход),
«Нет доверенности!» (обход), паспорт с черепом (прыжок), фейковая ссылка ИИ
(прыжок), «ШТРАФ» (прыжок), неактуальная практика (подкат). Штраф 18–25%.
Бусты: обложка паспорта, пропуск в суд, идеальное дело, блестящая речь,
проверенная практика, курс «ИИ для юриста».
Декор: колонны, скамьи, двери «Зал №3», стойка помощника судьи, расписание
заседаний, «Тишина, идёт заседание», «Посторонним вход воспрещён», красная дорожка.

## Честность волн
Волна = «ворота» на 3 дорожках. Блокирующих препятствий ≤ 2 на волну; минимум
одна дорожка свободна или проходима прыжком/подкатом. На лёгкой фазе гарантируется
полностью свободная дорожка. Базовый интервал волн 0.85–1.2 с (по уровню), к финалу
уровня сжимается до ~84% базового (минимум ~0.71 с на уровне 3) — всё равно в разы
больше времени смены полосы (0.15 с), честная траектория существует всегда.

## Цели уровня (информационные; победа — по дистанции)
1: собрать бусты 20, избежать задержек 40, множитель x2.
2: собрать бусты 25, избежать задержек 55, множитель x3.
3: собрать бусты 30, избежать задержек 70, множитель x3.

## Экраны
Старт → Выбор уровня (3 карточки, блокировка/статусы) → Игра (HUD: таймер,
уровень, счёт, множитель, дистанция, шкала уверенности с отметками 75/90,
защита, цели, круглый индикатор ускорения) → Пауза / Победа / Поражение →
после 3-го уровня Финал «Вы успели в суд» с итоговым счётом.

## Персонаж
Процедурный юрист: голова, волосы, костюм, галстук, руки/ноги, ботинки, портфель.
Процедурная анимация бега (махи рук/ног, покачивание), наклон при подкате,
поза при прыжке, мягкая тень-блоб.

## Критерии готовности
- Проект открывается в Unity 2021.3+ без ошибок компиляции.
- Меню «До суда 15 минут → Создать или обновить сцену» создаёт Main.unity;
  Play в этой (или даже пустой) сцене запускает игру со стартового экрана.
- Работают 3 дорожки, прыжок, подкат, пауза, перезапуск.
- 3 уровня с разными палитрами/декором/препятствиями/бустами; финальный экран.
- Уверенность управляет ускорением/защитой/поражением; победа по дистанции.
- Никаких внешних ресурсов, боссов, мини-игр, веб-технологий.

## Ограничения прототипа
- Модели и звук процедурные; фоновой музыки нет (только SFX).
- Отражения пола имитируются гладкостью материалов и бликами, без Reflection Probe.
- Референсы из ./References на момент реализации отсутствовали (папка пуста) —
  визуал построен по текстовому описанию; изображения можно положить в
  `./References` для следующей итерации.
