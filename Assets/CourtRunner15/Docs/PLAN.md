# PLAN — этапы реализации «До суда 15 минут»

## 0. Разведка (выполнено)
- Рабочая папка пуста: Unity-проекта нет → создаём структуру с нуля.
- Папки References нет → визуал по текстовому описанию из ТЗ.
- Unity Editor на машине не найден → batchmode-проверка невозможна;
  компиляция валидируется через csc + стабы Unity API (см. этап 6).
- dotnet SDK нет, есть классический csc (C# 5) → весь код в синтаксисе C# 5.

## 1. Каркас проекта
- Packages/manifest.json (модули: ugui, ui, physics, particlesystem, audio, imgui).
- ProjectSettings/ProjectVersion.txt (2022.3 LTS; подойдёт любой 2021.3+).
- Структура Assets/CourtRunner15 по ТЗ; README с инструкцией запуска.

## 2. Ядро (уровень 1 играбелен)
- GameState.cs — перечисления состояний/типов.
- LevelCatalog.cs — данные 3 уровней (баланс, палитры, препятствия, бусты, цели, вывески).
- GameManager.cs — машина состояний, счёт/комбо/уверенность/защита/ускорение,
  Esc/R, связывание всех систем, EnsureExists() + автозапуск в любой сцене.
- LevelManager.cs — дистанция, таймер, цели.
- PlayerController.cs — дорожки, прыжок, подкат, ручные AABB-столкновения.
- PlayerVisual.cs — процедурная анимация юриста.
- SpawnManager.cs — честные волны, пул объектов, бусты, подсчёт «избежал».
- WorldSegmentManager.cs — зацикленные сегменты мира.

## 3. Три уровня
- Палитры/туман/свет per-level (LevelVisualBuilder.ApplyGlobalLook).
- Темы сегментов: офис / улица / суд + дальний фон-задник.
- Разные наборы препятствий и бустов, баланс штрафов по уровню.
- Прогрессия: победа открывает следующий уровень (PlayerPrefs), финал после 3-го.

## 4. Визуал
- ProceduralModelFactory: юрист, все препятствия/бусты, декор (столы, здания,
  колонны, люди, фургон, банкомат, рамка, пристав…), русские вывески (TextMesh).
- MaterialFactory: Standard/URP-Lit автоопределение, emission, глянец,
  прозрачные лучи (Sprites/Default + градиентные текстуры), окна зданий.
- Эффекты: частицы сбора/удара, линии скорости, след, лучи бустов, конфетти,
  вспышка удара, тряска и FOV камеры.
- UI: тёмные полупрозрачные панели, крупный русский текст, шкала уверенности
  с отметками, круглый индикатор ускорения, 7 экранов.

## 5. Сцена и запуск
- Editor-меню «До суда 15 минут/Создать или обновить сцену» → Main.unity.
- Заранее записанный Main.unity (камера, свет, bootstrap) + .meta с
  фиксированным GUID для bootstrap-скрипта.
- Страховка: [RuntimeInitializeOnLoadMethod] — Play в любой сцене запускает игру.

## 6. Проверки
- csc (C# 5) + стабы Unity API в scratchpad: компиляция всех скриптов
  (синтаксис + межфайловая согласованность). Стабы в проект не попадают.
- Editor/CourtRunnerValidator.cs: меню-проверка + метод для Unity batchmode
  (-executeMethod CourtRunner15.EditorTools.CourtRunnerValidator.RunBatch).
- Свежий subagent: сверка реализации со SPEC.md/PLAN.md по требованиям.
- Честный финальный отчёт: что проверено, что нет (Unity недоступен).
